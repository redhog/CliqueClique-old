CliqueClique by redhog

A distributed/decentralized communication and editing environment

CliqueClique has a rather grandiose goal: To implement something of an extensible mix between a wiki, a forum and chat, without a central server. To simplify quite a bit, the protocol can be viewed as Internet News, where every message is its own newsgroup, containing just the comments to that message. Each node then subscribe to any of these "newsgroups", that is, messages, that that node is interested in comments to. This means that, as opposed to how it works in NNTP, no node needs to subscribe to all of comp.os.linux just to get the answers to that single question about locking in filesystem drivers... For more information please read doc/specification.txt This repository contains the core protocol implementation for the message distribution mechanism.
